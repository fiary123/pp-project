# 系统评估与演示指南 (毕业设计参考)

## 1. 核心演示场景说明 (Demo Scripts)

### 场景 A：智能营养再规划 (主打亮点：数据闭环)
- **操作描述**: 
  1. 为宠物狗狗生成初始营养方案。
  2. 进入反馈页面，提交“体重增加明显、排便正常”反馈。
  3. 点击“方案优化”。
- **展示重点**: 看到 AI 自动调减了 10% 的每日卡路里，并给出“置信度为 98%”的建议。

### 场景 B：高风险分诊识别 (主打亮点：医疗安全)
- **操作描述**: 
  1. 在分诊输入框输入：“狗狗误食了巧克力，一直在抖，呼吸很快”。
  2. 点击分析。
- **展示重点**: 系统立即识别出“极高风险（Emergency）”，并强烈建议立即联系线下兽医，体现系统的“守门员”价值。

### 场景 C：管理员审计溯源 (主打亮点：可解释性)
- **操作描述**: 
  1. 登录管理员账号。
  2. 进入“智能执行日志”管理页面。
- **展示重点**: 看到刚才分诊请求的详细执行日志，包括响应时长、调用的 Agent 名称以及是否使用了兜底逻辑，证明系统不是“黑盒”。

## 2. 系统局限性与未来工作 (用于论文结论 & 答辩应对)

### 局限性 (Limitations)
- **语义解析深度**: 目前基于 Pydantic 的结构化输入对自由文本的语义拆解仍依赖 LLM 的解析准确度，极端口语化表达可能影响精确配算。
- **知识库实时性**: 本地 ChromaDB 向量库的知识更新存在滞后，无法即时抓取最新的宠物医疗论文成果。
- **物种支持有限**: 核心算法目前精细支持犬、猫，对鸟类、爬行类等特殊宠物的营养模型尚不完善。

### 未来工作 (Future Work)
- **多模态融合**: 引入图像识别技术，实现通过拍摄宠物照片自动估算其 BCS（身体状况评分），减少人工录入误差。
- **私有云微调**: 尝试在特定宠物医疗语料集上微调 Llama/Qwen 等模型，以提升特定病症分诊的临床准确率。
- **地域化医疗对接**: 结合地理信息系统（GIS），在分诊发现高风险时，自动推荐距离用户最近的 24 小时急诊宠物医院。

## 3. 常见答辩问题预演 (QA Ready)

**Q: 为什么不直接调用大模型，而要搞这么多 Agent？**
A: 单一大模型在处理复杂长链路任务（如营养规划）时容易产生幻觉。通过 Agent 拆分职责，可以将计算逻辑、检索逻辑和反馈优化逻辑解耦，提高系统的专业性和可调试性。

**Q: 如果大模型 API 挂了，你的系统还能用吗？**
A: 本系统设计了完备的 **Fallback (降级)** 机制。我们在核心服务层内置了基于经典营养学公式（RER/DER）和关键词匹配的规则引擎。一旦大模型响应超时或失败，系统会自动切换到本地规则，保证基本的养护建议不中断。
