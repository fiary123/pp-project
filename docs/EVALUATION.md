# 系统评估与演示指南 (毕业设计参考)

## 1. 核心演示场景说明 (Demo Scripts)

### 场景 A：智能营养再规划 (主打亮点：数据闭环)
- **操作描述**:
  1. 登录普通用户账号，进入”营养规划”页面。
  2. 填写宠物信息（品种、体重、活动量），生成初始营养方案。
  3. 进入反馈页面，提交”体重增加明显、排便正常”反馈。
  4. 点击”方案优化”。
- **展示重点**: AI 根据反馈自动调减每日卡路里，并给出带”置信度”评分的新方案，体现**多智能体闭环优化**能力。

### 场景 B：高风险分诊识别 (主打亮点：医疗安全)
- **操作描述**:
  1. 进入”智能分诊”页面。
  2. 输入：”狗狗误食了巧克力，一直在抖，呼吸很快”。
  3. 点击分析。
- **展示重点**: 系统立即识别”极高风险（Emergency）”，强烈建议立即联系线下兽医，体现系统的”守门员”价值与**医疗安全设计**。

### 场景 C：管理员审计溯源 (主打亮点：可解释性)
- **操作描述**:
  1. 登录管理员账号（`admin@test.com`）。
  2. 进入”智能执行日志”管理页面。
- **展示重点**: 每条 AI 请求均有详细的执行日志，包括响应时长、Agent 名称、是否触发 Fallback，证明系统**不是黑盒**，具备完整的可解释性与审计能力。

### 场景 D：安全机制验证 (主打亮点：工程规范)
- **操作描述**:
  1. 不登录直接访问 `/api/posts`（POST）。
  2. 登录普通用户后尝试删除他人帖子。
  3. 普通用户调用 `/api/announcements`（POST）。
- **展示重点**: 系统返回 `401 未登录` / `403 无权限` 的精确错误，展示 **RBAC 权限控制** 和**资源归属验证**的完整实现。

## 3. 系统局限性与未来工作 (用于论文结论 & 答辩应对)

### 局限性 (Limitations)
- **语义解析深度**: 目前基于 Pydantic 的结构化输入对自由文本的语义拆解仍依赖 LLM 的解析准确度，极端口语化表达可能影响精确配算。
- **知识库实时性**: 本地 ChromaDB 向量库的知识更新存在滞后，无法即时抓取最新的宠物医疗论文成果。
- **物种支持有限**: 核心算法目前精细支持犬、猫，对鸟类、爬行类等特殊宠物的营养模型尚不完善。
- **消息实时性**: 当前聊天功能使用轮询方案（3秒间隔），在高并发场景下效率低于 WebSocket，可作为未来优化方向。

### 未来工作 (Future Work)
- **多模态融合**: 引入图像识别技术，实现通过拍摄宠物照片自动估算其 BCS（身体状况评分），减少人工录入误差。
- **私有云微调**: 尝试在特定宠物医疗语料集上微调 Llama/Qwen 等模型，以提升特定病症分诊的临床准确率。
- **地域化医疗对接**: 结合地理信息系统（GIS），在分诊发现高风险时，自动推荐距离用户最近的 24 小时急诊宠物医院（已预留高德地图工具接口）。
- **实时消息**: 将聊天功能升级为 WebSocket，降低延迟并减少服务器轮询压力。

## 4. 常见答辩问题预演 (QA Ready)

**Q: 为什么不直接调用大模型，而要搞这么多 Agent？**
A: 单一大模型在处理复杂长链路任务（如营养规划）时容易产生幻觉。通过 Agent 拆分职责，可以将计算逻辑、检索逻辑和反馈优化逻辑解耦，提高系统的专业性和可调试性。每个 Agent 的输入输出都有明确的 Schema 约束，减少了不确定性。

**Q: 如果大模型 API 挂了，你的系统还能用吗？**
A: 本系统设计了完备的 **Fallback（降级）** 机制。核心服务层内置了基于经典营养学公式（RER/DER）和关键词匹配的规则引擎。一旦大模型响应超时或失败，系统会自动切换到本地规则，并在日志中记录 `fallback_used=true`，保证基本养护建议不中断。

**Q: 你的系统有哪些安全措施？**
A: 系统实现了多层安全防护：① JWT + bcrypt 的标准认证体系，密码从不明文存储或比对；② RBAC 三级权限控制，所有写操作均验证登录状态和资源归属；③ 文件上传通过魔数检测真实类型，防止伪装绕过；④ 限流保护防止暴力破解；⑤ 所有 API 密钥通过环境变量管理，不在代码中硬编码。

**Q: 你怎么保证代码的正确性？**
A: 项目包含专项单元测试（`tests/test_security.py`），覆盖密码验证、JWT 安全、文件类型检测、营养计算边界等核心逻辑，共 14 个测试用例全部通过。同时有 `eval_engine.py` 对多智能体系统进行性能基准评估。
